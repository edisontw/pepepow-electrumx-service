Gateway Safety Boundary

Read-Only Architecture

The gateway serves public block data. It does not contain private keys, mnemonics, or signing systems, making it secure against credentials theft.

No Custody or Ledgers

We do not operate an invoice database. The payment monitor checks real-time address states. Reusing addresses is discouraged for clarity.

Private ElectrumX Integration

Our ElectrumX server is isolated and bound only to the local interface (`127.0.0.1`), routing all public requests through FastAPI rate limiters.

Public API Documentation

Developers can integrate with PEPEW Light using the following clean, JSON-RPC backed REST endpoints:

GET /api/health
Gateway status heartbeat

Purpose

Lightweight ping to verify the API Gateway is online and reachable. Does not communicate with ElectrumX.

Example request

curl -s https://light.pepepow.net/api/health

Response fields

Field Type Description
okbooleanTrue if gateway is healthy.
appstringApplication identifier.
envstringGateway run environment (e.g. production).
versionstringCurrent package version.
GET /api/status
ElectrumX node state & sync height

Purpose

Checks connections to ElectrumX server, reports sync height, protocol versions, and latency.

Example request

curl -s https://light.pepepow.net/api/status

Response fields

Field Type Description
okbooleanTrue if ElectrumX connection was checked.
envstringRuntime environment.
electrumx.connectedbooleanTrue if successfully handshake completed with the node.
electrumx.heightinteger | nullCurrent synced block height.
electrumx.response_time_msfloatRound-trip node latency in milliseconds.
errorstring | nullSanitized error code if connection is offline.

Safety note

Returns HTTP 200 even on connection loss to guarantee clean error formatting. Check electrumx.connected and the error field.
GET /api/address/{address}
Address balance & metadata

Purpose

Inspects balances (confirmed/unconfirmed) and transaction counts for a specified PEPEPOW address.

Example request

curl -s https://light.pepepow.net/api/address/PRfbEeHAKKbz6Voz85WJudrJwTA3ZbHunb

Response fields

Field Type Description
okbooleanRequest success state.
addressstringNormalized, validated address.
scripthashstringComputed scripthash identifier.
balance.confirmedintegerConfirmed balance in satoshis.
balance.unconfirmedintegerUnconfirmed mempool balance in satoshis.
history_countintegerTotal count of transactions for the address.
GET /api/address/{address}/history
Paginated history lookup

Purpose

Paginates through address transaction hashes. Supports offset and limit query variables.

Example request

curl -s "https://light.pepepow.net/api/address/PRfbEeHAKKbz6Voz85WJudrJwTA3ZbHunb/history?limit=5&offset=0"

Response fields

Field Type Description
historyarrayList of items containing tx_hash and block height.
has_morebooleanTrue if additional pages are available beyond the current offset.
limitintegerPage limit configuration (max 500).
offsetintegerCurrent offset pointer.
GET /api/tx/{txid}
Transaction hash lookup

Purpose

Placeholder endpoint designed to return transaction confirmations and block data in a subsequent phase.

Example request

curl -s https://light.pepepow.net/api/tx/453c4f723b2eb5cf4b8ef9818fac50d08ddc8ed05bc972ee464f2dbc55b6a3db

Response status

Returns HTTP 501 Not Implemented with code transaction_lookup_unavailable.
GET /api/payment/check
Address-level payment verifier

Purpose

Evaluates if a specific address has received at least the requested amount of PEPEW. Includes support for custom confirmations requirements and expiry timestamps.

Example request

curl -s "https://light.pepepow.net/api/payment/check?address=PRfbEeHAKKbz6Voz85WJudrJwTA3ZbHunb&amount=1"

Response fields

Field Type Description
statusstringStatus string: waiting, seen_in_mempool, partial, paid_unconfirmed, paid_confirmed, or overpaid.
requested_satsintegerRequested amount converted to satoshis.
total_received_satsintegerSum of confirmed and mempool satoshis.
confirmations_requiredintegerTarget confirmation threshold.
expiredbooleanTrue if current server time exceeds the expiry limits.

Safety note

This endpoint checks address-level totals only. Generate unique, fresh addresses per payment request to prevent ambiguous detection.
